# ================================================================
# CrushFlow V2 — Guía de Instalación y Deployment en Hostinger
# ================================================================

## Requisitos
- Node.js 18+ (Hostinger Business Web Hosting)
- MySQL 8.0+
- npm 9+

---

## 1. Preparar la base de datos en Hostinger

1. En el panel → **Bases de datos MySQL** → Crear BD
   - Nombre: `crushflow_v2` (o el que prefieras)
   - Crear usuario y asignar todos los permisos

2. Conectar al panel de phpMyAdmin e importar los scripts en orden:
   ```
   scripts/01-schema.sql
   scripts/02-seed.sql
   ```

## 3. Configuración de la app en Hostinger

En **Node.js App → Configuration**:

| Campo | Valor |
|-------|-------|
| Application Root | `/` |
| Application URL | `tudominio.com` |
| Application startup file | `dist/index.js` |
| Install Command | `npm install` |
| Build Command | `npm run build` |
| Start Command | `node dist/index.js` |

## 4. Deploy

```bash
# Local: compilar TypeScript
npm run build

# Verificar que dist/ se generó correctamente
ls dist/

# Subir al repositorio
git add .
git commit -m "deploy: CrushFlow V2"
git push origin main

# En Hostinger: hacer Restart/Redeploy
```

## 7. Arquitectura

**Backend:** Hexagonal / Clean Architecture
- `domain/` → Entidades puras
- `application/services/` → Lógica de negocio
- `interfaces/` → HTTP (Express controllers + routes)
- `infrastructure/` → MySQL (repositorios)

**Frontend:** SPA vanilla JS (sin frameworks)
- Sin build step — archivos estáticos servidos por Express
- Compatible con todos los navegadores modernos
- Responsivo: PC, tablet y móvil

**Seguridad:**
- JWT con refresh tokens (8h access / 7d refresh)
- RBAC granular por módulo y acción (`modulo.accion`)
- Rate limiting: 20 intentos de login / 15min; 300 req/min general
- Helmet (headers de seguridad)
- Bcrypt para contraseñas (10 rounds)
- Validación con Zod en todos los endpoints
- Soft delete (nunca eliminar datos reales)

**Multi-departamento:**
- 8 módulos: Dashboard, Inventario, Movimientos, Compras, Ventas, RRHH, Finanzas, Reportes
- 8 roles predefinidos, expandibles
- Permisos por módulo + acción o wildcards (`compras.*`, `*`)
- Auditoría automática de acciones
